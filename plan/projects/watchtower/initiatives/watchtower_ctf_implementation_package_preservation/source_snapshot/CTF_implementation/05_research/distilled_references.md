# Distilled References

## ATT&CK

- use ATT&CK to anchor tactic naming and cross-links, not as the sole source of offensive-security workflow design;
- pin the ATT&CK version used by the pack as ATT&CK v18 unless a later reviewed version is intentionally adopted;
- record that the October 28, 2025 ATT&CK v18 release introduced the asset-centric defensive model and retired older detection-centric assumptions;
- do not rely on deprecated ATT&CK surfaces without recording that they are deprecated.

## OWASP WSTG

- use WSTG as the main community-canonical reference for web testing workflow organization and coverage ideas;
- cite specific WSTG versions or sections rather than moving `stable` links;
- prefer explicit WSTG v4.2 section references or repository-tagged references until a later version is reviewed deliberately.

## NIST SP 800-115

- use it to shape operator workflow sequencing, engagement structure, and closeout/reporting expectations;
- keep it as workflow guidance, not as a substitute for pack-specific CTF runtime behavior.

## NIST SP 800-86

- use it to justify evidence capture discipline, provenance notes, and artifact handling expectations;
- adapt the forensic rigor proportionally for CTF use without losing chain-of-custody reasoning where evidence matters.

## NIST SP 800-92 Rev.1 Draft

- treat it as informative guidance for event-stream and audit design;
- do not present it as final binding authority.

## Diátaxis

- use it to keep operator guides, standards, references, and explainers distinct;
- do not let documentation structure blur machine-contract authority.
