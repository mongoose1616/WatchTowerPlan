# Dependency Register

| Dependency | Type | Why It Matters | Status |
|---|---|---|---|
| current shared pack commands | upstream contract | scaffold, bootstrap, validate, export, release proof | required |
| current shared governance surfaces | upstream contract | pack context, status/actor vocabulary, path rules | required |
| `plan` reference pack | implementation pattern | broad pack settings, manifest, and workflow surface examples | informative |
| `oversight` reference pack | implementation pattern | document-semantics, workflow metadata, compact pack patterns | informative |
| ATT&CK / WSTG / NIST research anchors | external guidance | taxonomy, workflow, evidence, and audit implications | required_for_docs |

## Notes

- all external guidance must remain subordinate to the live shared-core contract;
- research is for better standards and guides, not for overriding current repo truth.
