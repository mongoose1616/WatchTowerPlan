| `decision.pentest_pack_split` | whether future pentest support remains under `offensive_security/` or becomes a new hosted pack | keep one offensive-security pack for v1 | this is product-scope expansion, not a blocker for the first CTF pack |
| `decision.saved_query_views` | whether saved query views land in v1 and how they should evolve later | defer beyond v1; when added, ship pack-owned defaults first, user-local later, and keep user-local views outside the governed pack root | the workshop resolved the eventual direction, but real usage should prove which views are worth standardizing |
| `decision.provenance_review_impact_surface` | whether provenance and trust workflows gain stronger downstream review tooling | defer beyond v1; when added, prefer provenance-triggered re-review, downstream impact analysis, and a reviewer-facing affected-downstream view while flagging accepted artifacts for review rather than auto-invalidating them | this is valuable governance tooling, but not a baseline blocker for the first pack |
