## Quick Reference or Distilled Reference
### AI RMF Functions
| Function | Role | Local use |
|---|---|---|
| Govern | set accountability and oversight | define owners, policy, and review triggers |
| Map | understand context, stakeholders, and intended use | make system boundaries and impacts explicit |
| Measure | assess risks and system behavior | choose evidence, evaluation, and monitoring |
| Manage | prioritize and treat risks | define controls, mitigations, and responses |

### Core Rules
- Use AI RMF as an organizing model for governance, not as a ready-made implementation checklist.
- Keep intended use, stakeholders, and harm scenarios explicit.
- Translate the functions into concrete local controls before claiming alignment.

### Common Pitfalls
- Treating the framework as complete policy by itself.
- Skipping measurement or oversight because the governance language feels high level.
