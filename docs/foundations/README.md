# `docs/foundations`

## Description
`This directory contains the repository foundation documents. Use it for the durable product, design-philosophy, standards-context, technology-direction, and narrative materials that shape the rest of the documentation and implementation surfaces. These files are the canonical foundation layer for the repository.`

## Audience Routes

| Audience | Start Here | Then Read | Why |
|---|---|---|---|
| Engineers and maintainers | `engineering_design_principles.md` | `engineering_stack_direction.md`, `repository_standards_posture.md`, `product_direction.md` | Use this route when shaping the shared system, control plane, workflow layer, and implementation direction. |
| Product owners and engineering leads | `product_direction.md` | `customer_story.md`, `engineering_design_principles.md`, `repository_standards_posture.md` | Use this route when defining product shape, scope, tradeoffs, and delivery order. |
| Designers | `customer_story.md` | `product_direction.md`, `engineering_design_principles.md` | Use this route when shaping the operator experience, user journey, and interaction model. |
| Customers, operators, and buyers | `customer_story.md` | `product_direction.md` | Use this route when the goal is to understand the user problem, value, and intended experience rather than internal governance detail. |
