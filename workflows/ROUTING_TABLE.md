# Routing Table

Use this table to select the minimum workflow modules required for a request. Always load `modules/core.md`, then load only the modules needed for the matched task type(s).

During a non-documentation workflow, if a documentation gap is discovered, keep the current module set and merge the smallest documentation route needed. Add the relevant documentation leaf module (`modules/documentation_generation.md` for new docs or `modules/documentation_refresh.md` for existing docs) plus any required shared phase modules from that route that are not already loaded. If the gap is not handled in the same change, record it as explicit follow-up work.

When a task's main risk is drift between implementation behavior and companion docs or machine-readable lookup surfaces, use the Documentation-Implementation Reconciliation route or merge `modules/documentation_implementation_reconciliation.md` into the active route rather than relying only on informal handoff checks.

When a task's main risk is drift between traced planning or governance artifacts and their companion trackers, family indexes, or unified traceability joins, use the Traceability Reconciliation route or merge `modules/traceability_reconciliation.md` into the active route rather than relying only on planning leaf workflows or handoff review.

When a task's main risk is drift between schema-backed governed artifacts and their companion schemas, examples, indexes, registries, or loader and validator assumptions, use the Governed Artifact Reconciliation route or merge `modules/governed_artifact_reconciliation.md` into the active route rather than relying only on generic validation.

When a request both distills external source material and asks for a new repository document, merge the Reference Distillation and Documentation Generation routes rather than treating either route as sufficient by itself.
